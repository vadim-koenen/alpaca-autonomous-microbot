# P2-038B Exit Counterfactual Simulator

## Purpose
The purpose of the P2-038B Timeout / Fee / Exit Counterfactual Simulator is to analyze the historical closed trades extracted by P2-037 and simulate how alternative exit policies (such as earlier timeouts, breakeven-plus-fees, and maker/taker fee modifications) would have impacted profitability. 

This simulator is **advisory only**. It does not alter any live strategy, risk caps, capital, or trading parameters.

## Inputs
- Canonical normalized journal exports (`reports/journals/export_*.json`) generated by P2-037.
- High-resolution price-path / OHLCV data from `logs/coinbase_price_path.csv` or `data/ohlcv` (if available).

## Price-path Feasibility Caveat
The normalized journal exports only provide exact entry and exit prices at their respective timestamps. They do not contain full intra-trade tick data. If sufficient price-path data is not found in the local repository logs or data directories, the simulator cannot definitively reconstruct what would have happened if a trade were held longer or exited earlier.
In this case, the simulator will cleanly downgrade its capabilities, skip path-dependent simulations, and produce a baseline feasibility report.

## Policies Supported
If adequate path data exists, the following counterfactual policies are modeled:
- 30-minute timeout
- 45-minute timeout
- 60-minute timeout
- 75-minute timeout
- 90-minute baseline timeout
- breakeven-plus-fees exit
- earlier fee-aware exit
- tighter stop-loss
- smaller take-profit
- TP-hit probability vs TP distance

## What is Simulated vs Not Simulated
- **Simulated**: Fee sensitivities, notional scaling impacts, baseline PnL metrics.
- **Not Simulated**: Live execution conditions, slippage dynamically changing, or network latencies.
- **Unavailable**: If path data is missing, all path-dependent policies (timeouts, MFE/MAE, breakeven triggers) are strictly marked as unavailable and not faked.

## Fee/Notional Interpretation
- The simulator tests scaling the assumed average notional (e.g. 10x) to determine linearly scaled net profitability.
- It tests a maker-fee assumption (0.4%) compared to the baseline taker-fee assumption (0.6%) to analyze whether a maker-only policy would flip a net loss into a net win.

## How to Run
Run the simulator using:
```bash
python3 scripts/p2_038b_exit_counterfactual_simulator.py
```
The output will be saved deterministically in `reports/diagnostics/p2_038b_exit_counterfactual_simulator_YYYYMMDDTHHMMSSZ.json`.

## How to Interpret Insufficient Path Data State
If `path_data_status` is `insufficient` or `unavailable`, it means the repository lacks the localized tick data required to reconstruct intra-trade price behavior. In this state, only `baseline` and `notional_sensitivity`/`maker_taker_sensitivity` are calculated using the raw entry/exit prices. All other policies will be listed under `unavailable_policies` and skipped to prevent reporting fabricated or "faked" simulations.

## Acceptance Criteria
Per P2-038B senior-review bundle, the following metrics are explicit acceptance criteria.

**Required when path data is available:**
- per-policy gross PnL
- estimated fees
- net PnL
- fee-adjusted win rate
- notional sensitivity
- maker/taker assumptions
- MFE/MAE distributions
- TP-hit probability vs TP distance
- fee-inclusive breakeven per symbol
- intra-candle TP-vs-SL ambiguity bounds
- conservative/worst-case TP-vs-SL assumption when candle order is ambiguous

**Required when path data is unavailable:**
- do not fake path-dependent policies
- mark unavailable policies explicitly
- still compute baseline economics and fee/notional sensitivity from entry/exit prices
- explicitly state what price-path/OHLCV data is missing
