# Going to real money + weekly notifications (2026-06-16)

Two new capabilities: **Level-2 notifications** (weekly reminder) and a **gated live mode** (real $10).
Live is OFF by default and behind several explicit gates. Nothing trades real money until you
deliberately enable it AND confirm.

## What the bot does / does NOT decide

- **Fixed, human-chosen basket** (Conservative: SPY 35 / GLD 25 / SLV 15 / QQQ 15 / BTC 10). The bot
  does **not** pick assets, rotate, or time the market from "research" or news. It only splits each
  contribution by your weights and steers toward underweights. News is advisory/risk-only.
- To change the basket you edit `app_config.json` deliberately. That's *your* allocation decision.

## Level 2 — weekly notifications (no trading)

```bash
# send one now (test):
./run_app.sh --notify
# schedule it every Monday 09:00:
cp com.vadim.accumulator-weekly.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.vadim.accumulator-weekly.plist
```
It notifies you to review & approve, and surfaces any news RISK alert. It never places an order.

## Going live (real $10) — the gates

Real-money execution requires **all** of these, by design:
1. `live_trading_enabled = true` in `app_config.json` (set via the command below).
2. An explicit `--i-understand-real-money` confirmation on the command.
3. Reachable live broker (your live `ALPACA_API_KEY` — already in `.env`).
4. Per-contribution **dollar cap** (`live_max_contribution`, default $100) — a fat-finger guard.
5. No `runtime/ACCUMULATOR_STOP` kill-switch file present.

### Recommended sequence (after a couple of clean paper fills)

```bash
cd ~/Documents/Claude/Projects/Investing/alpaca-autonomous-microbot
# 1) enable live mode (turns OFF paper; real money armed but nothing traded yet)
./run_app.sh --enable-live --i-understand-real-money
# 2) place ONE real contribution (deliberate, gated):
./run_app.sh --go-live --i-understand-real-money
```
`--go-live` submits the plan ($10 → SPY/GLD/SLV/QQQ/BTC by weight) to your **live** Alpaca account.
Fund the account first (deposit, or set Alpaca's own recurring ACH) — the bot deploys existing cash,
it never pulls from your bank.

### Emergency stop (instant)

```bash
touch runtime/ACCUMULATOR_STOP        # blocks all live execution immediately
rm runtime/ACCUMULATOR_STOP           # resume
./run_app.sh --disable-live           # back to paper/sim
```

## Before you flip live — the honest checklist

- You ran paper and saw orders **fill correctly** (one crypto fill + one ETF fill during market hours).
- The Performance panel matched your Alpaca paper account.
- Funding model decided: you deposit to Alpaca (or Alpaca recurring ACH); the bot never touches your bank.

Notes: `runtime/STOP_TRADING` (the retired Coinbase bot's switch) stays present and untouched — the
accumulator uses its own `ACCUMULATOR_STOP`. Keys stay in `.env`, never printed/committed. The $100
default cap means even a bug can't spend more than $100 in one contribution; lower it if you like
(`live_max_contribution`).
